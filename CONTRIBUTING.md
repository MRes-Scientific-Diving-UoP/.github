# Contributing

Welcome! These are the shared working conventions for repositories in the
**MRes Scientific Diving** organisation. They apply to student project repos,
shared resources, and anything else in the org.

## New here?

Start with the **[Getting-Started](https://github.com/MRes-Scientific-Diving-UoP/Getting-Started)**
repo. It covers:

- [Connecting to GitHub with SSH](https://github.com/MRes-Scientific-Diving-UoP/Getting-Started/blob/main/guides/01-ssh-setup.md)
- [Creating your first repository](https://github.com/MRes-Scientific-Diving-UoP/Getting-Started/blob/main/guides/02-create-your-first-repo.md)
- [Structuring a project the tidy-R way](https://github.com/MRes-Scientific-Diving-UoP/Getting-Started/blob/main/guides/03-tidy-r-project-structure.md)

Starting a new project? Use the
**[project-template](https://github.com/MRes-Scientific-Diving-UoP/project-template)**
repo — click *"Use this template"*.

## Working conventions

- **Commit little and often**, with messages that describe *what* changed
  (`Add site-A quadrat data`, not `update`).
- **Never edit raw data by hand** — clean it in a numbered script.
- **Don't commit** large files (> ~50 MB), raw personal/sensitive data, or
  credentials. Use `.gitignore`.
- **Pull before you push** to avoid conflicts.
- Follow the [tidyverse style guide](https://style.tidyverse.org/) for R code.

## Getting help

Open an issue using one of the templates (supervision update / analysis help),
or email the programme team for anything personal or admin-related.
